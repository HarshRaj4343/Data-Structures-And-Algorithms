# Contributing

Thanks for contributing.

## Guidelines

1. Keep each solution focused on a single problem or topic.
2. Place new C++ files inside the correct folder under `src/cpp/`.
3. Use descriptive, lowercase file names with underscores when creating new files.
4. Do not commit local binaries, `.dSYM` folders, or editor-specific files.
5. If you add a new topic or project, update `README.md` to reflect it.

## Before Opening a Pull Request

1. Compile the changed C++ files locally.
2. Run any related scripts if you touched Python code.
3. Check that only source files and useful project assets are staged.
